How kexec works in XSM-Py
Kernel execution "Kexec" is a mechanism of the Linux kernel that allows "live" booting of a new kernel "over" the currently running kernel. kexec skips the bootloader stage (hardware initialization phase by the firmware or BIOS) and directly loads the new kernel into memory, where it starts executing immediately. This avoids the long times associated with a full reboot, and can help systems to meet high-availability requirements by minimizing downtime.

Source: Wikipedia.org

XSM-Py implements kexec in shell and GUI client, you can easily select a Linux based operating system you want to instant boot-in. At day 17.07.2011 xsm is a first graphical user interface implementing easy and quick access to kexec technology for all users at mass scale.

List of operating systems
To create functional list of available installed operating systems XSM uses libpybootutils - bootloaders configuration parser.
If you want to see supported bootloaders list you may visit Support for other bootloaders subpage.


Getting current kernel path
Current path where kernel image of currently booted OS is placed can be taken from /proc/cmdline, its usually first parametr BOOT_IMAGE.
Kexec removes BOOT_IMAGE parametr while booting new kernel.
So after first "Quick reboot" xsm adds xsm_image variable to kernel command-line containing path of currently loaded kernel.
This trick allows xsm remeber where is placed currently booted kernel even if kexec removes BOOT_IMAGE variable.

Example of command created and executed by xsm: 

kexec /boot/linux-3.0 --command-line="root=/dev/sda1 ro quiet xsm_image=/boot/linux-3.0"
