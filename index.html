X-Session Manager (XSM-PY)
X-Session Manager is a Python implementation of session management in X11.
It allows you to select desktop environment you want to work in (if login manager provides these functions too). XSM is a first graphical user interface with implemented kexec, it allows you to reboot machine without reinitializing hardware (skipping BIOS).

License: This software is distributed on Open and Free Software terms - GPLv3
Requirements: Python 2.x, PyGTK, GTK, X11, alang-py
Optional requirements: kexec, libpybootutils
Source code: xsm-py

To download project files you need a Git, then you can execute: 

$ git clone git://github.com/webnull/xsm-py
To install you just need to run ./install.sh as root: 
$ sudo ./install.sh

Predefined sessions are stored in /usr/share/xsm/sessions 

$ ls /usr/share/xsm/sessions/
default  dev-shell  kde4  openbox

Example of session:

$ cat /usr/share/xsm/sessions/openbox
#!/bin/bash
#openbox &
boxbg restore
lxpanel &
exec ck-launch-session openbox

Users can have also custom sessions in their $HOME/.xsm files

$ cat /home/webnull/.xsm
#!/bin/sh
pidgin &
skype &
exec ck-launch-session startkde

To exit session you dont have to run GTK version of /usr/bin/xsm-gui, you can use /usr/bin/xsm-exit

$ xsm-exit
## X-Session Manager ( Made by WebNuLL )
Available options: logout, shutdown, reboot, lock, kexec

$ xsm-exit lock
Going to sleep with xscreensaver...

Session Manager installation for specified user:

$ ln -s /usr/bin/xsm-init /home/webnull/.xinitrc
# You must link .xinitrc with /usr/bin/xsm-init to allow selecting sessions for specified users
